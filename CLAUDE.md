# communicate — 커뮤니케이션 스킬 동호회 슬라이드

## 프로젝트
공공기관(대학) 직원 대상 커뮤니케이션 스킬 향상 슬라이드 시리즈.
day01.html ~ day12.html 총 12개 파일로 완성 예정.

## 슬라이드 규격
- **포맷:** 16:9, 단일 HTML 파일 (파일당 슬라이드 1장)
- **폰트:** Pretendard CDN
- **레이아웃:** 좌측 38% 컬러 패널 + 우측 카드 **최대 2장** + insight bar
  - 카드 2장 초과 시 슬라이드 분리 (1파일 = 1주제)
- **빔 최적화:** `--s` CSS 변수로 뷰포트 기준 자동 스케일
- **네비게이션:** 하단 중앙 pill nav (현재 페이지 active)
  - ⚠️ **새 파일 추가 시 기존 모든 파일의 nav도 갱신** — day04 추가하면 day01~03 nav에 Day 04 링크 추가해야 함

## 새 슬라이드 제작 규칙

1. **`history.md` 확인** — 이미 다룬 주제 파악, 토론 결과와 대조 후 채택.
2. **`/productivity:persona-debate` 스킬 실행** — 페르소나 토론으로 주제 도출.
   - 샘플링 대상 직종: `공공행정 사무원`, `공공행정 전문가`, `국가행정 사무원`, `교육 및 훈련 사무원` 등 공공기관 계열
   - 필터 예시: `occupation IN ('공공행정 사무원', '공공행정 전문가', '국가행정 사무원', ...)`
3. **슬라이드 작성** — 토론에서 나온 꼭지 중 history 미등재 주제만 채택.
4. **`history.md` 업데이트** — 완성 후 주요 내용 기록.
5. **기존 파일 nav 갱신** — 전체 day*.html 파일의 nav에 새 링크 추가.

> **예외 — day02:** 페르소나 토론 미적용(기존 파일). 콘텐츠는 가족 간 소통으로 공공기관 업무 맥락과 다소 다름.
